# איפיון מערכת - "תן ברכה"

## מסמך דרישות מערכת (SRS)

**שם הפרויקט:** תן ברכה - Ten Bracha  
**תאריך:** יוני 2026  
**גרסה:** 1.0  

---

## 1. מבוא

### 1.1 מטרת המסמך
מסמך זה מגדיר את איפיון המערכת עבור אפליקציית "תן ברכה". המסמך מתאר את דרישות המערכת, הפונקציונליות, הארכיטקטורה, ממשק המשתמש והמגבלות של הפרויקט.

### 1.2 תיאור הפרויקט
"תן ברכה" היא אפליקציית ווב לזיהוי מזון וקביעת ברכה הלכתית. המשתמש מצלם או מעלה תמונה של מזון, מערכת בינה מלאכותית מזהה את סוג המזון, ומנוע כללים הלכתיים קובע את הברכה המתאימה בהתאם למסורת המשתמש (אשכנז, ספרד, עדות המזרח, חב״ד).

### 1.3 קהל יעד
ציבור דובר עברית שומר מצוות, המעוניין לדעת איזו ברכה לברך על מזון שלפניו, לרבות:
- אנשים המקפידים על הלכות ברכות
- צעירים ומבוגרים שאינם בקיאים בכל דיני הברכות
- אנשים הנתקלים במאכלים שאינם מוכרים להם

### 1.4 הגדרות ומונחים
| מונח | הגדרה |
|------|--------|
| ברכה ראשונה | הברכה הנאמרת לפני אכילת המזון |
| ברכה אחרונה | הברכה הנאמרת לאחר אכילת המזון |
| מסורת | מנהג הלכתי לפי עדה (אשכנז, ספרד, עדות המזרח, חב״ד) |
| שיעור | כמות מינימלית המחייבת ברכה אחרונה |
| ספק | מצב שבו קיימת אי-בהירות הלכתית לגבי הברכה |
| AI | בינה מלאכותית - מודל gemini-2.5-pro לזיהוי תמונות |

## 2. תיאור כללי

### 2.1 סקירת המערכת
המערכת בנויה כאפליקציית ווב Full-Stack הכוללת:
- **Frontend**: ממשק משתמש אינטראקטיבי בעברית (RTL)
- **Backend**: שרת API עם מסד נתונים ושירותי AI
- **AI**: מודל ראייה מרובת-מודאליות לזיהוי מזון מתמונות

### 2.2 תרשים זרימה כללי
```
משתמש ←→ מסך בחירת מסורת ←→ מסך העלאת תמונה
                                        ↓
                              שרת AI (זיהוי מזון)
                                        ↓
                              מסך תוצאות זיהוי
                                        ↓
                         ┌───────────────┴───────────────┐
                         ↓                               ↓
              מסך ברכה (ודאות)              מסך ספק (הפניה לקו חם)
```

### 2.3 הנחות יסוד
1. המשתמש דובר עברית
2. קיים חיבור לאינטרנט
3. למשתמש יש מצלמה או גישה לגלריית תמונות
4. המערכת אינה מחליפה פוסק הלכה - במקרה ספק מפנה לרב

## 3. דרישות פונקציונליות

### 3.1 בחירת מסורת (FR-01)
**תיאור:** המשתמש יבחר את המסורת ההלכתית שלו מתוך 4 אפשרויות:
- אשכנז
- ספרד
- עדות המזרח
- חב״ד

**קלט:** בחירה מתוך כרטיסיות אינטראקטיביות  
**פלט:** שמירת המסורת ב-session לשימוש בכל המסכים  
**הערה:** ספרד קרוב יותר לאשכנז מאשר לעדות המזרח מבחינת הלכתית

### 3.2 העלאת תמונת מזון (FR-02)
**תיאור:** המשתמש יוכל לצלם או להעלות תמונה של מזון  
**קלט:** תמונה ממצלמה או מגלריה  
**פלט:** תמונה נשלחת לשרת לזיהוי  
**מגבלות:**
- פורמטים נתמכים: JPG, PNG, WEBP
- גודל מקסימלי: 10MB
- תמיכה ב-Drag & Drop

### 3.3 זיהוי מזון באמצעות AI (FR-03)
**תיאור:** מודל AI (gemini-2.5-pro) מזהה את סוג המזון מהתמונה  
**קלט:** תמונת מזון  
**פלט:** רשימת תוצאות עם רמת ביטחון (confidence level)  
**API:** `POST /api/v1/food/recognize`  
**תגובה:**
```json
{
  "results": [
    {
      "food_name": "apple",
      "hebrew_name": "תפוח",
      "confidence": 0.95,
      "category": "פרי העץ"
    }
  ]
}
```

### 3.4 אישור/תיקון זיהוי (FR-04)
**תיאור:** המשתמש יאשר או יתקן את תוצאת הזיהוי  
**קלט:** בחירת המזון הנכון מתוך רשימת הצעות  
**פלט:** מזון מאושר לצורך קביעת ברכה

### 3.5 קביעת ברכה (FR-05)
**תיאור:** המערכת תציג את הברכה הראשונה והאחרונה בהתאם למזון ולמסורת  
**קלט:** מזון מאושר + מסורת נבחרת  
**פלט:** ברכה ראשונה, ברכה אחרונה, והערות (כגון תנאי שיעור)  
**API:** `POST /api/v1/food/blessing`  
**תגובה:**
```json
{
  "food_name": "apple",
  "hebrew_name": "תפוח",
  "tradition": "אשכנז",
  "first_blessing": "בורא פרי העץ",
  "after_blessing": "בורא נפשות",
  "requires_clarification": false,
  "notes": "אם נאכל שיעור"
}
```

### 3.6 טיפול במקרי ספק (FR-06)
**תיאור:** כאשר קיים ספק הלכתי, המערכת תפנה את המשתמש לקו חם הלכתי  
**קלט:** מזון עם `requires_clarification: true`  
**פלט:** מסך ספק עם מספרי קווים חמים לפי מסורת  
**קווים חמים:**

| מסורת | מספר טלפון |
|-------|-------------|
| אשכנז | 072-215-2222 |
| ספרד | 02-652-5555 |
| עדות המזרח | *3030 |
| חב״ד | 077-225-1770 |

### 3.7 חיפוש מזון ידני (FR-07)
**תיאור:** המשתמש יוכל לחפש מזון באופן ידני ללא צילום  
**API:** `GET /api/v1/food/list`  
**פלט:** רשימת כל 71 פריטי המזון עם ברכות

#### מסך 2 - העלאת תמונה
- אזור Drag & Drop להעלאת תמונה
- כפתור צילום ממצלמה
- כפתור בחירה מגלריה
- תצוגה מקדימה של התמונה שנבחרה
- כפתור "זהה מזון"

#### מסך 3 - תוצאות זיהוי
- תמונת המזון שהועלתה
- רשימת תוצאות עם רמת ביטחון (אחוזים)
- אפשרות לבחור מזון אחר מהרשימה
- כפתור "המשך לברכה"

#### מסך 4 - ברכה
- שם המזון בעברית
- ברכה ראשונה (מודגשת, גדולה)
- שאלת כמות (האם אכל/שתה שיעור?)
- ברכה אחרונה (מופיעה לאחר מענה על שאלת הכמות)
- הערות נוספות

#### מסך 5 - ספק (הפניה לקו חם)
- הסבר: "קיים ספק הלכתי לגבי ברכה זו"
- מספר קו חם לפי מסורת המשתמש
- כפתור חיוג ישיר
- אפשרות לחזור ולנסות שוב
